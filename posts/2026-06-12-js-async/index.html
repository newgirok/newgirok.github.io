자바스크립트 비동기, 한 번에 정리

2026-06-12
JavaScript비동기Promiseasync-await

자바스크립트는 싱글 스레드입니다. 한 번에 하나의 일만 합니다. 그런데 어떻게 서버 요청을 보내면서 동시에 화면도 그릴 수 있을까요.

동기 vs 비동기

동기 (Synchronous)
───────────────────────────────────
[작업 A] → 완료 → [작업 B] → 완료 → [작업 C]
              ↑ 기다림         ↑ 기다림

비동기 (Asynchronous)
───────────────────────────────────
[작업 A] → [작업 B] → [작업 C]  ← 메인 스레드는 계속 진행
      ↓
   (위임)
      ↓
   완료 후 콜백 실행

동기는 순서대로 하나씩. 비동기는 오래 걸리는 일은 맡겨두고 다음으로 넘어갑니다.

이벤트 루프 — 비동기의 실체

자바스크립트가 비동기처럼 동작할 수 있는 이유는 브라우저(Web APIs) 가 대신 처리해주기 때문입니다.

┌─────────────────────────────────────────────────┐
│                    브라우저                       │
│                                                  │
│  ┌──────────────┐      ┌──────────────────────┐ │
│  │  Call Stack  │      │      Web APIs        │ │
│  │  (싱글스레드) │      │  (멀티스레드)         │ │
│  │              │      │  Timer, fetch,        │ │
│  │  JS 코드 실행 │ ───▶ │  DOM 이벤트 등        │ │
│  └──────┬───────┘      └──────────┬───────────┘ │
│         │                         │ 완료         │
│         │  이벤트 루프             ▼             │
│         │  (Call Stack가    ┌──────────────────┐ │
│         │   비면 큐에서      │   Task Queue     │ │
│         │   꺼내서 실행)     │  (setTimeout 등) │ │
│         │                   ├──────────────────┤ │
│         └◀──────────────── │   Job Queue      │ │
│                             │  (Promise, 우선순위↑)│
│                             └──────────────────┘ │
└─────────────────────────────────────────────────┘

실행 순서 규칙

  1. Call Stack이 비어있어야 큐에서 꺼낸다
  2. Job Queue(Promise)가 Task Queue(setTimeout)보다 우선순위가 높다
setTimeout(() => console.log('A'), 0)
Promise.resolve().then(() => console.log('B'))
console.log('C')

// 출력: C → B → A

setTimeout이 0ms여도 Promise보다 늦게 실행됩니다.

비동기 처리의 3단 진화

1단계 — 콜백 (Callback)

function getUser(id, callback) {
  setTimeout(() => {
    callback({ id, name: '김철수' })
  }, 1000)
}

getUser(1, (user) => {
  getPosts(user.id, (posts) => {
    getComments(posts[0].id, (comments) => {
      // 콜백 지옥 🔥
    })
  })
})

작업이 늘어날수록 코드가 오른쪽으로 밀립니다. 에러 처리도 각 단계마다 따로 해야 합니다.

2단계 — Promise

Promise 상태 흐름
─────────────────────────────────────
          ┌─── fulfilled (성공) ───▶ .then()
Pending ──┤
          └─── rejected  (실패) ───▶ .catch()
                                        ↓
                                     .finally() ← 항상 실행

function getUser(id) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve({ id, name: '김철수' })
    }, 1000)
  })
}

getUser(1)
  .then(user => getPosts(user.id))
  .then(posts => getComments(posts[0].id))
  .then(comments => console.log(comments))
  .catch(error => console.error(error))
  .finally(() => console.log('완료'))

중첩 대신 체이닝으로 평탄해집니다. 에러도 .catch() 하나로 처리합니다.

여러 요청을 동시에 보낼 때

// 순차 실행 — 느림 (1초 + 1초 + 1초 = 3초)
const user = await getUser(1)
const posts = await getPosts(user.id)

// 병렬 실행 — 빠름 (가장 느린 것 기준)
const [user, posts] = await Promise.all([getUser(1), getPosts(1)])

3단계 — async / await

async function loadData() {
  try {
    const user = await getUser(1)
    const posts = await getPosts(user.id)
    const comments = await getComments(posts[0].id)
    console.log(comments)
  } catch (error) {
    console.error(error)
  }
}

await는 Promise가 완료될 때까지 해당 함수 내부에서만 대기합니다. 메인 스레드는 블로킹되지 않습니다. 동기 코드처럼 읽히지만 비동기로 동작합니다.

실전 — Fetch API

// GET 요청
async function getPost(id) {
  const response = await fetch(`/api/posts/${id}`)
  if (!response.ok) throw new Error(`HTTP error: ${response.status}`)
  return response.json()
}

// POST 요청
async function createPost(data) {
  const response = await fetch('/api/posts', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data),
  })
  return response.json()
}

fetch()는 네트워크 오류가 아니면 reject하지 않습니다. response.ok로 4xx·5xx를 직접 체크해야 합니다.

정리

콜백    → 동작하지만 코드가 지저분해진다
Promise → 체이닝으로 정리되지만 then이 길어진다
async/await → 동기처럼 읽히고 에러도 try/catch로 통일

| | 콜백 | Promise | async/await | |---|---|---|---| | 가독성 | 낮음 | 중간 | 높음 | | 에러 처리 | 각 단계마다 | .catch() | try/catch | | 병렬 처리 | 어려움 | Promise.all | Promise.all + await | | 현재 표준 | ✗ | △ | ✓ |

비동기를 이해하는 핵심은 이벤트 루프 입니다. 코드가 어떤 순서로 실행되는지 알면 async/await의 await가 왜 블로킹이 아닌지, Promise가 왜 setTimeout보다 먼저 실행되는지 자연스럽게 따라옵니다.
