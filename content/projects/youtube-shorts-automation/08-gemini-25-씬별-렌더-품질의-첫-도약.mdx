처음 파이프라인을 돌렸을 때, 솔직히 영상이 좀 심심했다. 뉴스 텍스트를 읽어주는 TTS 위에 정적인 배경 이미지 하나 올린 게 전부였으니까. "자동화는 됐는데, 이걸 누가 보나" 싶었다.

이번 스프린트에서 두 가지를 한꺼번에 바꿨다. 모델 업그레이드, 그리고 렌더 방식의 전면 재설계.

## Gemini 1.5 → 2.5 Flash

모델을 올렸더니 스크립트 품질이 눈에 띄게 달라졌다. 문장이 더 자연스러워졌고, 무엇보다 이번에 추가한 `scenes` 배열 생성이 훨씬 정교해졌다.

```json
{
  "scenes": [
    { "keyword": "폭염 경보", "imageQuery": "heatwave city street summer" },
    { "keyword": "냉방비 급등", "imageQuery": "electricity bill energy cost" }
  ]
}
```

각 씬에 한국어 키워드와 Unsplash 검색어를 함께 뽑아내도록 프롬프트를 짰다. 1.5로는 검색어 품질이 들쭉날쭉했는데, 2.5에서는 꽤 쓸 만한 수준이 나온다.

## 씬별 이미지 + Ken Burns 효과

render-worker에서 이 `scenes` 배열을 받아서 Unsplash API로 이미지를 내려받고, FFmpeg `zoompan` 필터로 Ken Burns 효과를 입혔다.

```ts
const zoompan = `zoompan=z='min(zoom+0.001,1.3)':x='iw/2-(iw/zoom/2)':y='ih/2-(ih/zoom/2)':d=${durationFrames}:fps=30`;
```

씬 타이밍은 TTS 음성 길이 기준으로 쪼갰다. Whisper가 뽑아준 VTT 타임스탬프에서 각 씬이 시작되는 시점을 계산해 이미지 전환 타이밍에 맞췄다.

정적인 배경 하나로 버티던 영상이, 이제는 씬마다 관련 이미지가 부드럽게 줌인되면서 전환된다. 같은 파이프라인인데 체감 품질이 확 올라갔다.

아직 이미지 라이선스 처리나 Unsplash rate limit 대응은 숙제로 남아있다. 하지만 일단 "볼 만한 영상"이 나오기 시작했다는 것 자체가 이번 스프린트의 성과다.
