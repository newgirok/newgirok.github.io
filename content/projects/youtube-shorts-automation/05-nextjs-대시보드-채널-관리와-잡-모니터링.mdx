파이프라인을 다 만들어놨는데 터미널로만 잡 상태 확인하는 건 너무 불편하다. 어차피 채널도 여러 개 관리해야 하고, 잡이 실패했는지 성공했는지 한눈에 보고 싶었다. 그래서 Next.js 15로 대시보드를 붙였다.

## Google OAuth 로그인

아무나 접근하면 안 되니까 NextAuth로 Google OAuth를 붙였다. 설정 자체는 간단한데, 내 계정만 허용하도록 콜백에서 이메일을 한 번 걸러준다.

```ts
callbacks: {
  async signIn({ profile }) {
    return profile?.email === process.env.ALLOWED_EMAIL;
  },
},
```

이게 없으면 OAuth 앱이 공개되는 순간 아무나 로그인할 수 있어서 꼭 필요하다.

## 채널 관리

YouTube 채널을 여러 개 운영할 수 있게 채널 단위로 묶었다. 채널 목록 조회, 추가, 삭제가 전부다. NestJS API를 호출하는 Server Action 몇 줄로 끝났다.

잡 생성 폼도 채널에 붙어있다. 스크립트 주제만 입력하면 SQS에 메시지가 들어가고 파이프라인이 돌기 시작한다. 입력창 하나짜리 폼인데, 이게 전체 파이프라인의 트리거다.

## 잡 모니터링

가장 필요했던 부분. 잡마다 상태 카드를 보여주는데 `PENDING`, `PROCESSING`, `DONE`, `FAILED` 네 가지다.

```tsx
const statusColor = {
  PENDING: "bg-gray-100",
  PROCESSING: "bg-blue-100",
  DONE: "bg-green-100",
  FAILED: "bg-red-100",
};
```

FAILED가 빨간색으로 딱 뜨면 어디서 터졌는지 바로 보인다. 실시간 스트리밍까지는 안 했고 페이지 진입 시 fetch로 충분하다고 판단했다. 잡이 수백 개씩 쌓이는 구조도 아니니까.

shadcn/ui 덕분에 컴포넌트 스타일링에 시간을 거의 안 썼다. 중요한 건 UI가 아니라 파이프라인이니까, 딱 필요한 것만 만들고 끝냈다.
