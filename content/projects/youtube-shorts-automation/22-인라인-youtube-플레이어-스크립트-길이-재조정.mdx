대시보드에서 업로드된 영상을 확인하려면 매번 YouTube 링크를 따로 열어야 했다. 귀찮음이 쌓이다 보니 결국 손댔다.

## 썸네일 클릭 → 인라인 플레이어

잡 상세 페이지에서 썸네일을 클릭하면 iframe YouTube 플레이어로 그 자리에서 전환된다. 별거 아닌 것 같지만 UX 체감이 꽤 달라진다.

한 가지 신경 쓴 건 레이아웃 시프트다. 썸네일과 플레이어 컨테이너 크기를 고정해두지 않으면 클릭 순간 화면이 덜컥 밀린다. 컨테이너에 고정 높이를 잡아서 전환 전후로 공간 크기가 바뀌지 않게 처리했다.

```tsx
{isPlaying ? (
  <iframe
    src={`https://www.youtube.com/embed/${videoId}?autoplay=1`}
    className="w-full h-full"
    allow="autoplay"
  />
) : (
  <img src={thumbnail} onClick={() => setIsPlaying(true)} />
)}
```

## 스크립트 길이, 데이터 보고 다시 잡았다

초기에 Shorts 스크립트 길이를 280자로 제한했는데, 실제 업로드된 영상 시청 데이터를 보니 너무 짧았다. 영상이 40~45초 언저리에서 뚝 끊기는 느낌이 있었고, 시청 지속률도 그 지점에서 떨어졌다.

380자로 올렸더니 55~60초 언저리로 맞춰졌다. Shorts 알고리즘 상 60초 미만이 최적이라는 얘기가 많은데, 이제야 그 범위에 제대로 들어온 셈이다.

Gemini 프롬프트 레퍼런스도 카테고리별 예시를 기존 1개에서 3개로 늘렸다. 예시가 많을수록 출력 일관성이 올라간다는 걸 반복 테스트에서 확인했다.

## 자잘한 것

Windows 환경에서 워커 에러 메시지가 CP949로 깨지던 문제도 잡았다. 로그 보다가 알아볼 수 없는 문자열이 나오는 게 계속 거슬렸는데, 이참에 처리했다.

작은 변경들인데 모여서 운영 편의가 꽤 올라간 릴리즈였다.
