자막 싱크 문제는 처음부터 골칫거리였다.

스크립트에서 문장을 쪼개고, 전체 음성 길이로 나눠서 타이밍을 추정하는 방식으로 버텨왔는데... 솔직히 말하면 그냥 "대충 맞는 척"이었다. 문장이 길어지거나 발음이 뭉개지면 자막이 1~2초씩 밀려버렸다.

그러다가 edge-tts 소스를 뒤지던 중 발견했다.

```
await communicator.save(audio_path, vtt_path)
```

**edge-tts가 VTT 파일을 직접 출력한다.** 단어 단위로. 타이밍 포함해서.

이걸 왜 지금 알았나 싶었다. TTS 워커를 즉시 수정해서 MP3와 VTT를 동시에 S3에 올리고, subtitle-worker에 VTT 경로를 함께 전달했다.

subtitle-worker 쪽 변화가 더 드라마틱했다. 기존 추정 로직을 전부 걷어내고, VTT의 타이밍을 그대로 읽어서 자막 엔트리를 구성했다. 단, 너무 짧은 엔트리(예: 0.1초짜리 조사 하나)가 튀어나오는 경우가 있어서 `splitEntry` 단계에서 앞 엔트리에 병합하는 후처리를 추가했다.

```ts
if (entry.duration < MIN_DURATION_MS) {
  mergeIntoPrevious(entries, entry);
  continue;
}
```

결과는 체감이 즉각적이었다. 자막이 입술에 딱 붙는다. 추정이 아니라 실측값이니까 당연한 일이다.

교훈이라면 — 외부 라이브러리가 이미 필요한 데이터를 제공하고 있는지 먼저 확인하자. 직접 구현하기 전에.
