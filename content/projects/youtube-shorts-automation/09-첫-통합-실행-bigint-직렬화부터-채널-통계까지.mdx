처음으로 전체 파이프라인을 통합해서 돌려봤다. 당연히 바로 터졌다.

## BigInt가 JSON을 못 넘어간다고?

Prisma에서 YouTube 조회수 같은 큰 숫자를 `BigInt`로 저장하는데, 이걸 `JSON.stringify`로 직렬화하려는 순간 런타임 에러가 난다. JavaScript `BigInt`는 기본적으로 JSON 직렬화가 안 된다.

해결은 간단했다. `BigInt.prototype.toJSON`을 오버라이드해서 `Number`로 변환하게 만들었다.

```ts
BigInt.prototype.toJSON = function () {
  return Number(this);
};
```

앱 진입점에 한 줄 추가하면 끝. 조회수가 `Number.MAX_SAFE_INTEGER`를 넘는 채널이 생기면 그때 다시 고민하기로 했다. 지금은 이게 맞다.

## 채널 통계 동기화 — YouTube Analytics API 연동

`POST /channels/:id/sync` 엔드포인트를 만들었다. 호출하면 YouTube Analytics API에서 구독자 수, 총 조회수를 당겨와서 DB에 저장한다. 대시보드에서 채널을 전환할 때마다 이 엔드포인트를 자동으로 찌르도록 프론트에서 연결했다.

덕분에 채널 선택 순간 최신 통계가 실시간으로 화면에 뜬다. 새로고침 안 해도 된다는 게 생각보다 꽤 쾌적하다.

## NextAuth 타입 문제

웹 대시보드 빌드가 NextAuth 타입 불일치로 막혔다. 세션 타입을 확장하는 방식이 버전마다 조금씩 달라서 생기는 흔한 이슈인데, 일단 빌드가 통과되도록 임시로 완화했다. 기술 부채로 적어두고 넘어간다.
