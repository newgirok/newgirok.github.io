오늘은 "왜 재시작하면 데이터가 날아가지?" 와 "쓰레기 영상이 유튜브에 올라가는 거 막아야 해" 두 가지를 잡았다.

## Docker 재시작하면 DB가 초기화되는 문제

로컬에서 `docker compose restart` 한 번 했더니 Prisma 마이그레이션이 처음부터 다시 돌고 있었다. 순간 멘탈이 나갔다.

원인은 단순했다. `docker-compose.yml`에서 Postgres 볼륨이 anonymous volume으로 선언되어 있었던 것.

```yaml
# 이전 — 컨테이너 재생성 시 날아감
volumes:
  - /var/lib/postgresql/data

# 이후 — named volume으로 영속성 확보
volumes:
  - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

anonymous volume은 컨테이너가 내려가면 GC 대상이 된다. named volume은 명시적으로 `docker volume rm` 하기 전까지 살아있다. 기본 중의 기본인데 놓쳤다.

## 쓰레기 영상 업로드 방지

렌더 워커가 가끔 이상한 파일을 뱉는 경우가 있다. 길이가 0초이거나, 해상도가 뭔가 이상하거나. 그걸 그냥 유튜브에 올리고 있었다는 게 문제였다.

업로드 워커에 ffprobe 검증 단계를 끼워 넣었다.

```ts
const probe = await ffprobe(filePath);
const video = probe.streams.find(s => s.codec_type === 'video');

if (!video || probe.format.duration < MIN_DURATION) {
  logger.warn('품질 기준 미달, 업로드 건너뜀');
  return;
}
```

해상도, 코덱, 길이를 체크해서 기준 미달이면 업로드 큐에서 그냥 드랍한다. 어설프게 올리는 것보다 안 올리는 게 낫다.

## 기타

SB Aggro 폰트가 자막에 안 먹히는 문제도 있었는데, `fc-cache` 등록 순서 문제였다. Dockerfile에서 폰트 복사 후 `fc-cache -fv`를 실행해야 하는데 순서가 뒤바뀌어 있었다. 여러 번 시도 끝에 잡았다. 별거 아닌데 시간을 꽤 썼다.
