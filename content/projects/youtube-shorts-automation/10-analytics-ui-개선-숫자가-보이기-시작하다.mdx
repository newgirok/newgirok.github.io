파이프라인을 다 만들었는데 정작 "얼마나 잘 되고 있는지"를 볼 수가 없었다. 영상이 올라가는 건 보이는데 조회수가 얼마나 나오는지, 시청 시간은 어떤지 — 그냥 YouTube 스튜디오 직접 들어가서 확인하고 있었다. 대시보드를 만들어놓고 YouTube 스튜디오를 따로 켜는 게 말이 되나.

## YouTube Analytics API 연동

YouTube Data API랑 Analytics API는 다르다. Data API는 영상 메타데이터(조회수, 좋아요)고, Analytics API는 시청 시간·RPM 같은 수익화 지표다. 둘 다 따로 붙여야 한다.

```ts
// Analytics API — 채널 단위 시청 시간, RPM 수집
const analyticsRes = await youtube.reports.query({
  ids: `channel==${channel.youtubeChannelId}`,
  metrics: 'estimatedMinutesWatched,estimatedRevenue',
  dimensions: 'day',
  startDate, endDate,
});
```

DB도 손봤다. `ChannelAnalytics`에 `watchTimeMinutes` 컬럼 추가, `Job`에는 `privacyStatus` 추가. 업로드 후 private → public 전환 상태를 추적하기 위해서다. 쇼츠는 업로드 직후 processing 상태라 잠깐 private으로 잡히는 경우가 있는데, 이걸 sync 때마다 갱신해두면 "왜 안 보이지?" 디버깅이 훨씬 쉬워진다.

조회수/좋아요는 잡 단위로도 실시간 동기화한다. 어떤 스크립트가 잘 되는지 보려면 채널 합산이 아니라 잡별 데이터가 필요하다.

## UI는 그냥 부서져 있었다

솔직히 말하면 처음부터 반응형을 제대로 안 잡았다. 모바일에서 채널 카드가 넘쳐흐르고, Job 상세 페이지는 상태 배지가 레이아웃 밖으로 튀어나갔다.

한 번에 다 고쳤다 — 채널/홈 페이지 그리드 반응형, Job 상세 페이지 레이아웃, 상태 표시 개선까지. 그리고 썸네일 문제가 하나 있었는데, `mqdefault`(320px)로 불러오다 보니 Shorts 썸네일이 흐릿하게 나왔다. `maxresdefault` 먼저 시도하고 실패하면 `hqdefault`로 fallback하는 걸로 바꿨다. 이게 생각보다 체감이 크다.

숫자가 보이기 시작하면 만드는 맛이 달라진다. 어떤 주제가 조회수가 높은지, 시청 시간이 긴 건 어떤 포맷인지 — 이제부터가 진짜 최적화 구간이다.
