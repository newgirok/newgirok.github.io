업로드 자동화를 만들면서 한 가지를 계속 미뤄두고 있었다. 바로 YouTube 설명란이다.

영상은 자동으로 올라가는데 설명란은 빈칸이거나 하드코딩된 더미 텍스트였다. 영상 내용과 전혀 무관한 설명란을 달고 올리는 건 좀 아니잖아. 결국 이것도 자동화해야 했다.

## Gemini가 스크립트 짤 때 설명란도 같이 뽑는다

기존엔 Gemini한테 스크립트 텍스트만 생성하도록 시켰는데, 어차피 뉴스 내용을 이미 다 읽은 상태잖아. 그 시점에 title, description, tags를 한꺼번에 뽑는 게 훨씬 자연스럽다. API 호출도 한 번이고.

```typescript
const result = await model.generateContent({
  contents: [{ role: 'user', parts: [{ text: prompt }] }],
});

// 응답에서 title / script / description / tags 동시 파싱
const { title, script, description, tags } = parseScriptResponse(result);
```

description은 세 파트로 구성했다.

1. 뉴스 해설 요약 — 영상에서 다룬 핵심 내용 2~3줄
2. 해시태그 — tags 배열을 `#태그` 형식으로 변환
3. AI 생성 공시 문구 — "이 영상은 AI가 생성한 콘텐츠입니다" 류의 고정 문구

세 번째가 은근히 중요하다. YouTube 정책상 AI 생성 콘텐츠는 명시 의무가 생기는 방향으로 가고 있어서, 미리 박아두는 게 맞다고 판단했다.

## 업로드 워커는 그냥 쓰면 된다

script-worker가 생성한 description을 S3에 메타데이터로 저장해두면, 업로드 워커는 그걸 꺼내서 YouTube API에 그대로 넘긴다. 별도 로직 없음. 이게 파이프라인 구조의 장점이기도 하다. 각 워커가 자기 역할만 하면 된다.

이제 뉴스 수집부터 YouTube 업로드까지 사람 손이 닿는 지점이 없다. 진짜 완전 자동화가 됐다.
