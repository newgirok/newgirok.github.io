처음엔 어필리에이트 마케팅 콘텐츠로 시작했다. "AI가 상품 리뷰 스크립트 만들고 자동 업로드하면 수익 나오지 않나?" 하는 단순한 아이디어였는데, 실제로 만들어보니 콘텐츠 품질이 처참했다. 뻔한 광고 문구를 TTS로 읽어주는 영상이 무슨 가치가 있겠어.

그래서 방향을 틀었다. **뉴스/시사 해설 콘텐츠**로.

뉴스는 소재가 매일 자동으로 공급된다. 공익성도 있고, AI 해설 형식이 Shorts에도 자연스럽게 맞는다. 스크립트 프롬프트를 어필리에이트 문구 생성기에서 뉴스 해설 형식으로 전면 교체했다.

```
// 이전: 상품 특징 나열 → 구매 유도
// 이후: 사건 요약 → 맥락 설명 → 핵심 포인트 1~2개
```

API 쪽엔 뉴스 자동 수집 잡을 트리거하는 엔드포인트도 추가했다. 스케줄러가 주기적으로 찔러주면 알아서 기사 긁어오고 파이프라인 태우는 구조.

한 가지 놓쳤던 게 있었는데, **YouTube AI 생성 콘텐츠 공시 의무**다. 2024년부터 정책이 바뀌어서 AI로 만든 영상엔 명시적으로 표기해야 한다. 업로드 로직에 `containsSyntheticMedia: true` 플래그와 공시 문구를 추가했다. 안 하면 채널 제재 맞을 수도 있으니 이건 선택이 아니라 필수.

ESM 전환은... 그냥 고통이었다. `require`가 먹히던 패키지들이 죄다 ESM-only로 넘어가면서 tsconfig 여기저기 손봐야 했다. 특별한 이야기는 없고 그냥 해야 하는 일이었다.

피벗하고 나서 콘텐츠 방향이 훨씬 명확해진 느낌이다. 시스템은 그대로고 프롬프트 하나 바꿨을 뿐인데 결과물이 완전히 달라졌다는 게 AI 파이프라인의 묘한 점이기도 하고.
