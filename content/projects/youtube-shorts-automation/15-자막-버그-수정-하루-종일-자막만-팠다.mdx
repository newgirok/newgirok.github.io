오늘은 기능 추가 없이 하루를 다 썼다. 자막만.

처음엔 "금방 고치겠지" 했는데 커밋이 6개가 됐다. revert까지 포함해서.

## 특수문자가 자막을 망친다

자막에 마침표, 쉼표, 물음표가 그대로 붙어 나오면 화면에서 생각보다 많이 거슬린다. Whisper가 뱉은 텍스트를 그대로 쓰고 있었으니 당연한 결과였다.

```ts
text = text.replace(/[.,?!]/g, "");
```

단순한 정규식인데 이게 없으니까 자막이 뉴스 기사처럼 보였다. Shorts에는 간결한 게 맞다.

## 폰트 이름이 틀렸다

ASS 자막 파일에 폰트명을 `SB Aggro Bold`로 쓰고 있었는데 렌더 결과물에서 폰트가 적용이 안 됐다. FFmpeg는 PostScript 내부 이름으로 폰트를 찾는다. 시스템에 설치된 폰트의 실제 이름을 확인해보니 달랐다.

```ass
Style: Default,SB Aggro Bold,56,...
```

이걸 PostScript 이름 기준으로 수정하고 나서야 렌더에서 의도한 폰트가 나왔다. 폰트 이름 하나 때문에 몇 번을 렌더 돌렸는지 모르겠다.

## 제목 선행 낭독은 실패였다

자막이 너무 짧게 끊기는 문제를 해결하려다가 "TTS가 제목을 먼저 읽으면 어떨까?" 하는 아이디어를 붙였다. 그런데 막상 음성을 들어보니 스크립트 첫 문장이 이미 제목을 이어받아 시작하는 구조라 제목이 두 번 반복됐다.

바로 revert했다. 아이디어가 틀렸으면 빨리 되돌리는 게 낫다.

대신 자막 길이 제한 쪽으로 방향을 바꿨다. 한 자막 블록에 텍스트가 너무 길면 잘라내는 방식으로 처리했다. Shorts는 화면이 좁으니까 두 줄 이상 넘어가면 오히려 가독성이 떨어진다.
