AI가 생성한 텍스트를 그냥 쓰면 어딘가 어색하다. 특히 뉴스를 요약해서 YouTube Shorts 스크립트로 바꿀 때, 그 "어색함"이 쌓이면 채널 전체 퀄리티가 무너진다.

이번 작업은 거창한 기능 추가가 아니라, 그 어색함을 하나씩 잡아내는 과정이었다.

## "~라고 합니다"는 이제 그만

가장 먼저 건드린 건 description 문체였다. Gemini가 뉴스 기사를 요약할 때 습관처럼 "~라고 합니다", "~라고 전했습니다" 같은 간접 인용체를 썼다. 뉴스 앵커처럼 들려야 하는데, 누군가한테 전달받은 얘기를 다시 전달하는 느낌이 났다.

프롬프트에 명시적으로 박았다. "직접 서술체로 쓸 것. ~했습니다, ~입니다로 종결할 것."

## "이라고"가 두 번 나오면 뭔가 잘못된 거다

픽스 중에서 제일 사소해 보이지만 체감은 컸다. "A라고 밝혔다고 합니다" 같은 문장이 간간이 나왔다. 읽다가 멈추게 되는 구조다. 프롬프트에 규칙으로 못 박았다. "이라고" 계열 어미는 한 문장에 연속으로 쓰지 말 것.

## "여러분은 어떻게 생각하세요?"는 CTA가 아니다

comment_bait 필드는 댓글 참여를 유도하는 질문을 넣는 칸인데, Gemini가 가끔 "여러분은 어떻게 생각하세요?" 같은 아무 맥락 없는 질문을 뱉었다. 콘텐츠와 연결된 구체적인 질문이어야 의미가 있다. 주제 없는 일반 질문 생성을 프롬프트 레벨에서 막았다.

## 프롬프트만 믿지 말고, 코드로 검증해라

제일 중요한 변경은 이거다. script 필드가 격식체(`~습니다`)로 끝나는지 코드 레벨에서 검증하도록 했다.

```ts
if (!script.endsWith("습니다.") && !script.endsWith("입니다.")) {
  // 재시도
}
```

프롬프트에 아무리 잘 써도 LLM은 가끔 규칙을 어긴다. "잘 써달라고 했으니까 잘 썼겠지"라고 믿는 순간 파이프라인이 무너진다. 검증은 코드가 해야 한다.

---

결국 이번 작업의 핵심은 **프롬프트는 의도를 전달하고, 코드는 결과를 보장한다**는 원칙을 하나씩 적용한 거다. 작은 픽스들이지만, 쌓이면 확실히 다르다.
