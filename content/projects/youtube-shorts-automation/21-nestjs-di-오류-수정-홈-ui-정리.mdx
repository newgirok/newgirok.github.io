## tsx로 NestJS 돌리면 안 되는 이유

개발 서버 켜자마자 DI 오류가 터졌다.

```
Nest can't resolve dependencies of the VideoService (?). 
Please make sure that the argument at index [0] is available in the AppModule context.
```

분명히 `@Injectable()` 붙여놨는데. 코드 문제가 아니었다.

범인은 `tsx`였다. `tsx`로 TypeScript를 실행하면 NestJS가 의존하는 `reflect-metadata`의 데코레이터 메타데이터가 제대로 등록되지 않는다. NestJS DI는 런타임에 타입 정보를 읽어야 하는데, `tsx`가 그 타이밍을 어긋나게 만든다.

해결은 간단했다. `@swc-node/register`로 교체.

```json
// package.json
"scripts": {
  "start:dev": "node --require @swc-node/register src/main.ts"
}
```

SWC는 `reflect-metadata`를 올바른 순서로 처리해서 데코레이터가 정상 동작한다. 빌드 속도도 빠르니 일석이조.

## 홈 화면 채널 탭 제거

홈 화면에 채널 탭이 있었는데, 솔직히 거기 있을 이유가 없었다. 채널 관리는 채널 전용 페이지에서 하면 되고, 홈은 대시보드 역할에만 집중하는 게 맞다.

탭 하나 없앴을 뿐인데 화면이 훨씬 깔끔해졌다. 기능을 추가하는 것만큼 빼는 것도 중요하다는 걸 다시 느꼈다.
